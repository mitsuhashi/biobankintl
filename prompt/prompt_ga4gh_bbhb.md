## GA4GH_BBHB対応表作成および最終条文案作成プロンプト（再々修正版：論点単位の分割マッピング）

あなたは法務およびバイオバンク運営の専門家です。
以下の4つの入力ファイルに基づき、GA4GH DAAとバイオバンクハンドブック（BBHB）の条文対応表を作成し、さらに第1回レビュー会議の議論を反映した「最終条文案」を提案してください。

### 1. 入力ファイル
1.  **GA4GH条文ファイル**: `mta_ga4gh_ja.md` (GA4GH DAA 日本語版)
2.  **BBHB条文ファイル**: `mta_handbook.md` (バイオバンクハンドブック雛形)
3.  **議論ログファイル**: `ga4gh_ukbb.pdf` (UKBB対応表および第1回レビュー会議の議論録)
4.  **GA4GH英語条文ファイル**: `GA4GH DAA Template Clauses 2024 Final.pdf` (GA4GH DAA 英語版)

### 2. 処理ロジック

#### A. 条文のマッピング（比較・構造化・抜粋配置）
*   **基準構造**: GA4GHの条文構造（1.1, 1.2...）を基準に行を作成してください。
*   **BBHBの配置（関連部分の抜粋）**:
    *   BBHBの条文を読み込み、意味的に対応するGA4GHの条文の行に配置してください。
    *   **重要**: BBHBの条文が複数の要素を含む場合（例：第4条が「法令遵守」と「複製制限」の両方を含む場合）、全文を全ての行にコピーするのではなく、**そのGA4GH条文に直接関連する部分（特定の項、号、または一文）のみを抜粋**して配置してください。
    *   抜粋する際は、**原文の文言を変更・要約せず**、そのまま抜き出してください。
    *   対応するGA4GH条文がないBBHB固有の条文は、「BBHBのみ」として、文脈的に最も適切なセクションの末尾に行を挿入して配置してください。

#### B. GA4GH条文への英文併記
*   GA4GH条文セルには、`日本語条文<空行>英語条文` の形式で、GA4GH英語条文ファイルから対応する英文を抜粋して追記してください。

#### C. 第1回レビュー会議議論のマッピング（論点単位の分割）
`ga4gh_ukbb.pdf` の「第1回レビュー会議議論」列を参照し、以下のルールでマッピングしてください。

1.  **対象**: 「第1回レビュー会議議論」列が「なし」以外の行。
2.  **分割とマッピング**: 一つのセル内に複数の論点（「条文案タイトル＋条文テキスト＋会話ログ」のセット）が含まれている場合、**論点単位で分割**してください。
3.  **配置**: 分割した各論点を、内容的に最も関連するGA4GH条文の行に配置してください。
4.  **転記内容**: 各論点ブロックの内容（条文案と会話ログ）は、**一言一句変更せずそのまま**転記してください。
5.  **新規行**: 関連するGA4GH条文がない論点については、最も近いセクションに新規行を挿入して配置してください。

#### D. AI提案（論点と提案の統合）
*   **AI提案（自由提案）**: これまでの議論や比較を勘案し、**次回議論すべき論点（課題）**およびそれに対する推奨提案を記述してください。
*   **AI提案（UKBB取込提案）**: UKBBから取り込むべき内容を提案してください。
*   ※ GA4GHにない内容は、独立した条文（例：4.1.1）を追加する形で提案してください。

#### E. 判定および分析
*   **判定**:
    *   `同意`：内容が完全に一致。
    *   `混在`：同意点と相違点の両方が含まれる。
    *   `相違`：内容が相違している。
    *   `GA4GHのみ` / `BBHBのみ`
    *   `第1回議論`：議論ログから追加された行。
    *   `AI提案`：AIが新規に追加した行。
*   **同意点・相違点**: 【同意点】と【相違点】に分けて簡潔に記述。
*   **試料・情報区分**: `両方` / `試料のみ` / `情報のみ`

#### F. 最終条文案の作成（ハイブリッド方針）
1.  **構造維持**: 条文番号および構成はGA4GHを維持する。
2.  **文体維持**: 原則としてGA4GHの翻訳調の文言を使用する。
3.  **内容強化**: BBHB、UKBBの要素、AI提案を反映させる場合は、GA4GHの文言に対し**必要最小限の加筆・修正**を行う。
4.  **差分出力**: 原文からの変更点とその理由を記述。
5.  **レビュープラン**: レビュー時のアクションプランを箇条書きで提案。

### 3. 出力テーブル形式
以下の列を持つマークダウン表を出力してください。

| 列名 | 内容 |
|---|---|
| **GA4GH条項名** | 条項名（空欄時は「同上」） |
| **GA4GH番号** | 1.1, 1.2... |
| **GA4GH 条文** | 日本語全文 + 英語全文 |
| **試料・情報区分** | 両方／試料のみ／情報のみ |
| **判定** | 同意／混在／相違／GA4GHのみ／BBHBのみ／第1回議論／AI提案 |
| **同意点・相違点** | 分析コメント |
| **BBHB条文** | **関連部分の抜粋（原文のまま）** |
| **第1回レビュー会議議論** | **関連する論点ブロックのみ（原文のまま）** |
| **AI提案（自由提案）** | **次回論点および推奨提案** |
| **AI提案（UKBB取込提案）** | UKBBからの取込提案 |
| **最終条文案** | 作成した条文案 |
| **原文との差分** | 変更点と理由 |
| **最終条文案レビュープラン** | アクションプラン |

### 4. 制約事項
*   出力が長くなる場合は分割して出力してください。
*   BBHB固有の重要条項は、GA4GH構成内の適切な場所に新設してください。
*   セルに値がない場合は「なし」と出力してください。
