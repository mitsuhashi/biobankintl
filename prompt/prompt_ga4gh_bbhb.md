## GA4GH_BBHB対応表作成および最終条文案作成プロンプト

あなたは法務およびバイオバンク運営の専門家です。
以下の3つの入力ファイルに基づき、GA4GH DAAとバイオバンクハンドブック（BBHB）の条文対応表を作成し、さらに第1回レビュー会議の議論を反映した「最終条文案」を提案してください。

### 1. 入力ファイル
以下の情報を読み込んでください。
1.  **GA4GH条文ファイル**: `mta_ga4gh_ja.md` (GA4GH DAA)
2.  **BBHB条文ファイル**: `mta_handbook.md` (バイオバンクハンドブック雛形)
3.  **議論ログファイル**: `ga4gh_ukbb.pdf` (UKBB対応表および第1回レビュー会議の議論録)
4.  **GA4GH英語条文ファイル**: `GA4GH DAA Template Clauses 2024 Final.pdf` (GA4GH DAAの英語版)

### 2. 処理ロジック

#### A. 条文のマッピング（比較）
*   GA4GHの条文構造（1.1, 1.2...）を基準に行を作成してください。
*   BBHBの条文を読み込み、意味的に対応するGA4GHの条文の行に配置してください。対応するものがない場合は「BBHBのみ」として適切な位置（または末尾）に挿入してください。

#### B. 第1回レビュー会議議論のマッピング
`ga4gh_ukbb.pdf` の第1回レビュー会議議論列が「なし」以外の行を、以下の処理をしてロジックでAで作成した対応表（以下、出力対応表）に追加してください。

* `ga4gh_ukbb.pdf` のGA4GH条文番号と一致する出力対応表のGA4GH条文番号の行の第1回レビュー会議議論セルに`ga4gh_ukbb.pdf`第1回レビュー会議議論セルの値をそのまま全てコピーしてください。
* GA4GH条文番号が一致する行が見つからなければ、GA4GH条文番号に一番近い出力対応表のGA4GH条文番号の下に１行挿入して、その行のGA4GH条文番号と第1回レビュー会議議論の値を`ga4gh_ukbb.pdf` から出力対応表にコピーしてください。例：条文番号2.2が出力対応表にない場合は2.1の直ぐ下に挿入する。
* `ga4gh_ukbb.pdf`の第1回レビュー会議議論列が「なし」以外の行は、必ず出力対応表に含まれるようにしてください。

#### C. 最終条文案の作成（ハイブリッド方針）
議論およびBBHB/UKBBの要素を踏まえ、以下のルールで「最終条文案」を作成してください。
1.  **構造の維持:** 条文番号および構成はGA4GHを維持する。
2.  **文体の維持:** 原則としてGA4GHの翻訳調の文言をそのまま使用する。
3.  **内容の強化:** AI提案1、AI提案2及びBBHBやUKBBに必要な要素が含まれる場合、GA4GHの文言に**必要最小限の加筆・修正（括弧書きや一文の追加）**を行って反映させる。
4. 最終条文案とGA4GH原文との差分を原文との差分に出力する。
5. 最終条文案レビューする際にどういうアクションを取るべきかのアクションプランを最終条文案レビュープランに箇条書きで出力してください。

#### D. GA4GH条文に英文を追加

GA4GH条文を`日本語<空行>英語`のようにGA4GH英語条文ファイルから対応する英文を抜粋して追加してください。

#### E. AI提案

* **AI提案（自由提案）**: これまでの議論を勘案して特に制約なく推奨事項を提案してください。GA4GHにない内容は独立した条文を追加する形で提案してください。例えば、4.1に追加したい場合は、4.1.1を新規条文として作成してください。
* **AI提案（UKBB取込提案）**: UKBBから取り込むべき内容を提案してください。GA4GHにない内容は独立した条文を追加する形で提案してください。例えば、4.1に追加したい場合は、4.1.1を新規条文として作成してください。

#### F. 判定、同意点・相違点

* **判定**; GA4GHとBBHBの比較結果を以下の分類で出力すること。定義は以下の通り。
  * 同意：GA4GHとBBHB条文の内容が完全に一致している。
  * 混在：GA4GHとBBHB条文の内容に同意点と相違点の両方が含まれている。
  * 相違：GA4GHとBBHB条文の内容が完全に相違している。
  * GA4GHのみ: GA4GHだけに条文がある
  * BBHBのみ：BBHBだけに条文がある
  * 第１回議論：第１回レビュー会議で議論した内容を追加
  * AI提案：E.AIの提案で追加したUKBB条文
* **同意点・相違点**：【同意点】と【相違点】に分割して、簡潔に要点を記述すること。

#### G. 試料・情報区分
条文が試料、情報の一方に関連する条文か、両方に関連する条文かを判定してください。
* 試料のみ：試料（マテリアル）にのみ関係する条文
* 情報のみ：情報（データ）にのみ関係する条文
* 両方：試料にも情報にも関係する。両方に共通する条文。

### 3. 出力テーブル形式
以下の列を持つマークダウンの表を出力してください。

| 列名 | 説明 | 作成ルール/出力例 |
|---|---|---|
| **GA4GH条項名** | GA4GH条項名 | 1.定義、2.利用...  空欄の場合は「同上」と出力すること。|
| **GA4GH番号** | GA4GHの枝番 | 1.1, 1.2... |
| **GA4GH 条文** | GA4GHの全文 | 省略せずに全文字出力すること。 |
| **試料・情報区分** | 試料、情報の一方に関連する条文か、両方に関連する条文かの区分 | 両方／試料のみ／情報のみ |
| **判定** | GA4GHとBBHBの比較結果 | 同意／混在／相違／GA4GHのみ／BBHBのみ |
| **同意点・相違点** | 分析コメント | 【同意点】と【相違点】に分割して、簡潔に要点を記述。 |
| **BBHB条文** | BBHBの全文 | 条文番号も含めて省略せずに全文字出力すること。 |
| **第1回レビュー会議議論** | UKBB議論ログ | 上記ロジックBに基づき、関連する議論を全量転記。 |
| **第２回レビュー会議論点案** | AI提案 | 相違点や議論に基づき、次回議論すべき論点を箇条書き。 |
| **AI提案（自由提案）** | 自由記述 | これまでの議論を勘案してAIが推奨する提案。新規条文の追加を含む。  |
| **AI提案（UKBB取込提案）** | UKBBから取り込むべき内容 | UKBBから取り込むべき内容を提案する。新規条文の追加を含む。|
| **最終条文案** | **今回作成する案** | 上記ロジックCに基づき、GA4GHベースで修正した条文案。 |
| **原文との差分** | 変更点の説明 | 最終条文案についてGA4GH原文から何を変更・追加したか、その理由（「UKBB反映」「BBHB整合」等） |
| **最終条文案レビュープラン** |最終条文案レビューする際にどういうアクションを取るべきかのアクションプランをAIが提案|1.<<文言1>>を追加するべきか検討<br>2.<<文言2>>を削除するべきか検討|


### 4. 制約事項
*   一度に出力できない場合は分割して出力してください（「続きを出力」で対応します）。
*   条文テキストは「...」などで省略せず、必ず全文を出力してください。
*   BBHBにしかない重要条項（存続条項など）は、GA4GHの構成の中で最も適切な場所に新設（例：15.3）してください。
*   GA4GHとBBHBの条文は抜け漏れなく全て出力されていることを確認してください。
*   セルに出力する値がない場合は、「なし」と出力して、表の列がずれないようにしてください。
