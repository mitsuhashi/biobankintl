## GA4GH_BBHB対応表作成および最終条文案作成プロンプト

あなたは法務およびバイオバンク運営の専門家です。
以下の3つの入力ファイルに基づき、GA4GH DAAとバイオバンクハンドブック（BBHB）の条文対応表を作成し、さらに第1回レビュー会議の議論を反映した「最終条文案」を提案してください。

### 1. 入力ファイル
以下の情報を読み込んでください。
1.  **GA4GH条文ファイル**: `mta_ga4gh_ja.md` (GA4GH DAA)
2.  **BBHB条文ファイル**: `mta_handbook.md` (バイオバンクハンドブック雛形)
3.  **議論ログファイル**: `ga4gh_ukbb.csv` (UKBB対応表および第1回レビュー会議の議論録)
4.  **GA4GH英語条文ファイル**: `GA4GH DAA Template Clauses 2024 Final.pdf` (GA4GH DAAの英語版)

### 2. 処理ロジック

#### A. 条文のマッピング（比較）
*   GA4GHの条文構造（1.1, 1.2...）を基準に行を作成してください。
*   BBHBの条文を読み込み、意味的に対応するGA4GHの条文の行に配置してください。対応するものがない場合は「BBHBのみ」として適切な位置（または末尾）に挿入してください。

#### B. 議論ログのマッピング（高度な紐づけ）
`ga4gh_ukbb.csv` の議論内容を、以下のロジックでAで作成した対応表に追加してください。
*   **GA4GH条文番号の一致:** ga4gh_ukbb.csvのGA4GH条文番号とAで作成した対応表のGA4GH条文番号が一致すること。それ以外のマッチングはしないでください。
*   **全量表示:** 議論ログの内容は要約せず以下の内容を表示してください。
    * GA4GH条文番号/UKBB条文番号/UKBB条文
    * 発言者と発言内容のセットを全て表示してください。

#### C. 最終条文案の作成（ハイブリッド方針）
議論およびBBHB/UKBBの要素を踏まえ、以下のルールで「最終条文案」を作成してください。
1.  **構造の維持:** 条文番号および構成はGA4GHを維持する。
2.  **文体の維持:** 原則としてGA4GHの翻訳調の文言をそのまま使用する。
3.  **内容の強化:** BBHBやUKBBに必要な要素が含まれる場合、GA4GHの文言に**必要最小限の加筆・修正（括弧書きや一文の追加）**を行って反映させる。

#### D. GA4GH条文に英文を追加

GA4GH条文を`日本語<空行>英語`のようにGA4GH英語条文ファイルから対応する英文を抜粋して追加してください。


### 3. 出力テーブル形式
以下の列を持つマークダウンの表を出力してください。

| 列名 | 説明 | 作成ルール |
|---|---|---|
| **GA4GH番号** | GA4GHの枝番 | 1.1, 1.2... |
| **GA4GH 条文** | GA4GHの全文 | 省略せずに全文字出力すること。 |
| **判定** | 比較結果 | 同意／混在／相違／GA4GHのみ／BBHBのみ |
| **同意点・相違点** | 分析コメント | 簡潔に要点を記述。 |
| **BBHB番号** | 対応するBBHB枝番 | ない場合は「なし」 |
| **BBHB 条文** | BBHBの全文 | 省略せずに全文字出力すること。 |
| **第1回レビュー会議議論** | UKBB議論ログ | 上記ロジックBに基づき、関連する議論を全量転記。 |
| **第２回レビュー会議論点案** | AI提案 | 相違点や議論に基づき、次回議論すべき論点を箇条書き。 |
| **AI提案１** | 方針選択 | 以下の選択肢から推奨を選定：<br>1. 弁護士確認<br>2. GA4GH優先（文言調整のみ）<br>3. バンク選択式にする<br>4. 議論・修正案の提示<br>5. バンク個別方針確認 |
| **最終条文案** | **今回作成する案** | 上記ロジックCに基づき、GA4GHベースで修正した条文案。 |
| **原文との差分** | 変更点の説明 | GA4GH原文から何を変更・追加したか、その理由（「UKBB反映」「BBHB整合」等）。 |

### 4. 制約事項
*   一度に出力できない場合は分割して出力してください（「続きを出力」で対応します）。
*   条文テキストは「...」などで省略せず、必ず全文を出力してください。
*   BBHBにしかない重要条項（存続条項など）は、GA4GHの構成の中で最も適切な場所に新設（例：15.3）してください。
